Blueprint Implementation Details

Under the hood: How ManufacturingIQ leverages Spark Structured Streaming and MLflow for real-time factory intelligence.

1. Real-Time Tolerance Anomaly Detection (Spark Streaming)

Ingesting high-frequency IoT sensor data (vibration, temperature, micrometer readings) from CNC machines to detect out-of-tolerance deviations instantly.

from pyspark.sql.functions import from_json, col, window import mlflow.pyfunc # Load predictive maintenance/tolerance model from MLflow Registry model_uri = "models:/CNC_Tolerance_Anomaly/Production" anomaly_udf = mlflow.pyfunc.spark_udf(spark, model_uri) # Stream IoT data from Kafka/EventHubs streaming_df = spark.readStream \ .format("kafka") \ .option("kafka.bootstrap.servers", "factory-kafka.internal:9092") \ .option("subscribe", "cnc-telemetry-v1") \ .load() # Parse payload and apply ML model to detect deviations in real-time processed_stream = streaming_df.selectExpr("CAST(value AS STRING) as json_payload") \ .select(from_json(col("json_payload"), schema).alias("data")) \ .select("data.*") \ .withColumn("is_anomalous", anomaly_udf(col("vibration_hz"), col("temp_c"), col("spindle_rpm"))) # Write flags to Silver Delta Table to trigger alerts on the shop floor query = processed_stream.writeStream \ .format("delta") \ .outputMode("append") \ .option("checkpointLocation", "/mnt/delta/checkpoints/cnc_telemetry") \ .table("main.factory_silver.cnc_anomalies")
