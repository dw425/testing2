Implementation Details

Under the hood: How RiskIQ leverages PySpark, MLflow, and Unity Catalog to power these insights.

1. PII Detection Pipeline (Mosaic AI / PySpark)

RiskIQ uses automated data classification jobs that run against new Delta Lake tables to identify unmasked PII (SSNs, Emails, Phone numbers) using NLP libraries like Presidio integrated via PySpark UDFs.

from pyspark.sql.functions import udf, col from presidio_analyzer import AnalyzerEngine # Initialize NLP Analyzer analyzer = AnalyzerEngine() def detect_pii(text): if not text: return "None" results = analyzer.analyze(text=text, entities=["PHONE_NUMBER", "EMAIL_ADDRESS", "US_SSN"], language="en") return ",".join([r.entity_type for r in results]) # Register UDF for distributed execution detect_pii_udf = udf(detect_pii) # Apply to raw bronze table to flag risky columns df_bronze = spark.read.table("main.raw_data.customer_intake") df_flagged = df_bronze.withColumn("pii_detected", detect_pii_udf(col("free_text_notes"))) # Save to Silver layer for dashboard reporting df_flagged.filter(col("pii_detected") != "None").write.mode("append").saveAsTable("main.governance.pii_alerts")

2. Enhanced RBAC Auditing via Unity Catalog Logs

Standard RBAC restricts access, but RiskIQ analyzes the system.access.audit tables to establish behavioral baselines. If a user downloads 100x their normal volume, the ML model flags it as "High Vol Export".

import mlflow # Load Unity Catalog System Audit Logs audit_logs = spark.sql(""" SELECT event_time, user_identity, action_name, request_params.table_full_name FROM system.access.audit WHERE action_name IN ('generateTemporaryTableCredential', 'runCommand') """) # Load trained MLflow Anomaly Detection Model (Isolation Forest) model_uri = "models:/RiskIQ_Access_Anomaly/Production" anomaly_model = mlflow.pyfunc.load_model(model_uri) # Predict Anomalies on user behavior features features_df = extract_user_features(audit_logs) predictions = anomaly_model.predict(features_df) # Trigger Alert to Databricks SQL Dashboard and Slack if predictions.contains_anomaly: trigger_slack_alert(user=features_df.user_identity, risk_level="High")

3. GDPR Right-To-Be-Forgotten Execution

When compliance generates an audit report or executes a deletion request, RiskIQ uses Databricks Delta Lake's DELETE operations which natively support ACID compliance to prove to auditors the data was removed securely.

# Execute GDPR Deletion across Gold Customer Tables spark.sql(""" DELETE FROM main.gold.customer_profiles WHERE customer_id = 'USER_99421'; """) # Force Vacuum to remove stale data files permanently (proving compliance) spark.sql(""" VACUUM main.gold.customer_profiles RETAIN 0 HOURS; """) # Log execution for the Compliance & Audit Hub UI log_compliance_action(framework="GDPR", action="Data Subject Deletion", status="SUCCESS")
